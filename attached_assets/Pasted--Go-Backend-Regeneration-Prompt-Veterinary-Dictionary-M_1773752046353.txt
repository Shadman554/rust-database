# Go Backend Regeneration Prompt - Veterinary Dictionary Management System

## Overview
Regenerate the complete backend API for the VetStan Veterinary Dictionary Management System using Go programming language. This system manages veterinary medical data including dictionary terms, diseases, drugs, books, instruments, tests, slides, notes, normal ranges, and more.

## Current Architecture (Reference)
The existing system is built with Python/FastAPI and includes:
- 15+ data models with complex relationships
- Authentication system with JWT tokens
- 50+ API endpoints across multiple modules
- File upload capabilities for images
- Search and pagination functionality
- Local storage synchronization
- Multi-language support (Kurdish/English)

### 2. **Technology Stack**
- **Web Framework**: Gin (or Echo/Fiber)
- **Database**: PostgreSQL with GORM
- **Authentication**: JWT tokens with golang-jwt
- **Validation**: go-playground/validator
- **File Upload**: multipart/form-data support
- **Logging**: logrus or zap
- **Configuration**: Viper for environment variables
- **Documentation**: Swagger/OpenAPI with swaggo
- **Testing**: Go's built-in testing + testify

### 3. **Database Models (PostgreSQL + GORM)**

#### User Model
```go
type User struct {
    ID        uint      `gorm:"primaryKey" json:"id"`
    Username  string    `gorm:"unique;not null" json:"username"`
    Email     string    `gorm:"unique;not null" json:"email"`
    Password  string    `gorm:"not null" json:"-"`
    FirstName string    `json:"first_name"`
    LastName  string    `json:"last_name"`
    Role      string    `gorm:"default:'user'" json:"role"`
    IsActive  bool      `gorm:"default:true" json:"is_active"`
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
}
```
